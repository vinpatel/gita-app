<task type="auto">
  <name>Task 1: Create Astro 5 project with all dependencies</name>
  <files>package.json, astro.config.mjs, tsconfig.json</files>
  <action>
    1. Run `npm create astro@latest . -- --template basics --no-install --typescript strict` in the project root (gita-app/).
       - If the directory is not empty (due to .planning/ and .git/), the scaffolder may prompt. Use `--skip-houston` if needed, or create in a temp directory and move files.
       - Alternative: if the interactive scaffolder blocks, manually create the project files using the research patterns below.

    2. Install all Phase 1 dependencies in one command:
       ```
       npm install astro @astrojs/preact preact tailwindcss @tailwindcss/vite @fontsource/tiro-devanagari-sanskrit @fontsource/noto-serif-devanagari astro-pagefind
       ```
       Note: Do NOT install `zod` separately — use `astro/zod` re-export (Zod v3). Installing zod@4 directly causes type conflicts (see withastro/astro#14117).

    3. Configure `astro.config.mjs` exactly as follows:
       ```javascript
       import { defineConfig } from 'astro/config';
       import tailwindcss from '@tailwindcss/vite';
       import preact from '@astrojs/preact';

       export default defineConfig({
         site: 'https://placeholder.github.io',
         base: '/gita-app',
         output: 'static',
         integrations: [preact()],
         vite: {
           plugins: [tailwindcss()],
         },
       });
       ```
       - `site` and `base` are placeholders — CI/CD (Plan 01-03) will inject correct values via `configure-pages`.
       - Do NOT use `@astrojs/tailwind` — it is deprecated and does not support Tailwind v4.
       - Do NOT add a `tailwind.config.js` file — Tailwind v4 uses CSS `@theme {}` blocks instead.

    4. Configure `tsconfig.json`:
       ```json
       {
         "extends": "astro/tsconfigs/strict",
         "compilerOptions": {
           "jsx": "react-jsx",
           "jsxImportSource": "preact",
           "baseUrl": ".",
           "paths": {
             "@components/*": ["./src/components/*"],
             "@layouts/*": ["./src/layouts/*"],
             "@data/*": ["./src/data/*"]
           }
         }
       }
       ```

    5. Verify: `npm run build` succeeds with zero errors and produces a `dist/` directory.
  </action>
  <verify>
    <automated>cd /Users/vin/Development/gita-app && npm run build 2>&1 | tail -5 && test -d dist && echo "BUILD OK" || echo "BUILD FAILED"</automated>
  </verify>
  <done>package.json lists astro, @astrojs/preact, preact, tailwindcss, @tailwindcss/vite, font packages, and astro-pagefind. astro.config.mjs uses @tailwindcss/vite in vite.plugins and preact() in integrations. tsconfig.json extends astro/tsconfigs/strict with Preact JSX. `npm run build` produces dist/ with zero errors.</done>

    2. Create `src/styles/global.css`:
       ```css
       @import "tailwindcss";

       /* Design tokens for Gita app */
       @theme {
         --font-devanagari: 'Tiro Devanagari Sanskrit', 'Noto Serif Devanagari', serif;
         --font-size-devanagari-min: 1.25rem; /* 20px minimum per DES-03 */
       }
       ```
       - This is the Tailwind v4 pattern. No `@tailwind base/components/utilities` directives (that is v3).
       - No `tailwind.config.js` file (v4 uses CSS-native configuration).

    3. Create `src/layouts/BaseLayout.astro`:
       ```astro

       import '../styles/global.css';

       interface Props {
         title: string;
         description?: string;

       ---
       <!doctype html>
       <html lang="en">
         <head>
           <meta charset="UTF-8" />
           <meta name="viewport" content="width=device-width, initial-scale=1.0" />
           <meta name="description" content={description} />
           <title>{title} | Bhagavad Gita</title>
         </head>
         <body class="min-h-screen bg-white text-gray-900">
           <slot />
         </body>
       </html>
       ```
