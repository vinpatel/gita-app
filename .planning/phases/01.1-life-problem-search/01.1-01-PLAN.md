  <files>package.json, src/pages/search-index.json.ts</files>
  <action>
1. Install Fuse.js: `npm install fuse.js`

2. Create `src/pages/search-index.json.ts` — a static Astro JSON endpoint that runs at build time:
   - Import `getCollection` from `astro:content` and `APIRoute` type from `astro`
   - Export a `GET` function that:
     - Calls `getCollection('chapters')` to get all 701 verses
     - Maps each verse to a compact object: `{ id: string, c: number, v: number, q: string[], t: string[], e: string }`
       - `id` = `d.id` (e.g. "BG-1.1")
       - `c` = `d.chapter`
       - `v` = `d.verse`
       - `q` = flatten all 4 question arrays: `[...d.questions.personal_growth, ...d.questions.career_business, ...d.questions.health, ...d.questions.relationships]`
       - `t` = `d.themes`
       - `e` = `d.explanation.slice(0, 200)` (snippet for result display)
     - Returns `new Response(JSON.stringify(index), { headers: { 'Content-Type': 'application/json' } })`
   - This is a static endpoint (SSG mode) — `GET` is called once at build time, no `prerender` export needed

Do NOT add any caching headers (static file, served by CDN). Do NOT use `export const prerender = true` — Astro 5 static mode does this by default.

<task type="auto">
  <name>Task 2: Create LifeSearch Preact island with Fuse.js fuzzy search and ARIA combobox</name>
  <files>src/components/islands/LifeSearch.tsx</files>
  <action>
Create `src/components/islands/LifeSearch.tsx` — a Preact island component:

1. **Types**: Define `IndexEntry` interface matching the search index shape: `{ id: string; c: number; v: number; q: string[]; t: string[]; e: string; }`

2. **Fuse.js configuration**:
   - Keys: `{ name: 'q', weight: 3 }` (questions — highest priority), `{ name: 't', weight: 1 }` (themes)
   - `threshold: 0.4` (tighter than default 0.6 — fewer false positives per research)
   - `ignoreLocation: true` (questions can match anywhere in array)
   - `minMatchCharLength: 3`
   - `includeScore: true`
   - `shouldSort: true`

3. **Props**: `{ baseUrl: string }` — passed from Astro page, used to prefix fetch URL

4. **State**:
   - `query: string` — current search input
   - `results: Fuse.FuseResult<IndexEntry>[]` — search results
   - `isLoading: boolean` — true while index is being fetched
   - `activeIndex: number` — keyboard navigation index (-1 = none)
   - `isOpen: boolean` — whether results dropdown is visible
   - `fuseRef: useRef<Fuse<IndexEntry> | null>` — Fuse instance, created once

5. **Lazy index loading**:
   - `loadIndex()` — async function called on input `onFocus`
   - If `fuseRef.current` already exists, return immediately (no re-fetch)
   - Set `isLoading = true`, fetch `${baseUrl}search-index.json`, parse JSON, create `new Fuse(data, fuseOptions)`, store in `fuseRef.current`, set `isLoading = false`
   - Handle fetch errors gracefully (log to console, don't crash)

6. **Search handler**:
   - `handleSearch(value: string)` — called on `onInput`
   - Set `query = value`, reset `activeIndex = -1`
   - If `value.length < 3` or `fuseRef.current` is null, set `results = []` and return
   - Call `fuseRef.current.search(value, { limit: 8 })`, set results
   - Set `isOpen = results.length > 0`

7. **Keyboard navigation** (onKeyDown on the input):
   - ArrowDown: increment `activeIndex` (wrap to 0 at end)
   - ArrowUp: decrement `activeIndex` (wrap to last at start)
   - Enter: if `activeIndex >= 0`, navigate to that result's verse page
   - Escape: close dropdown, reset `activeIndex`

8. **ARIA attributes** (critical for accessibility per research pitfall 4):
   - Input: `role="combobox"`, `aria-autocomplete="list"`, `aria-expanded={isOpen}`, `aria-controls="search-results"`, `aria-activedescendant={activeIndex >= 0 ? `result-${activeIndex}` : undefined}`
   - Results container: `role="listbox"`, `id="search-results"`
   - Each result: `role="option"`, `id={`result-${index}`}`, `aria-selected={index === activeIndex}`

9. **Result item rendering**:
   - Each result is an `<a>` tag linking to `${baseUrl}verse/${item.c}/${item.v}/`
   - Display: verse reference (e.g. "Chapter 2, Verse 47"), explanation snippet (`item.e`), and up to 3 theme tags as small pills
   - Hover/focus styles using Tailwind classes consistent with the project's design (neutral/warm palette)

10. **Curated theme pills** (typeahead suggestions before typing):
    - When `query` is empty and input is focused, show 8 clickable theme pills: `['grief', 'anger', 'decision-making', 'anxiety', 'purpose', 'duty', 'fear', 'attachment']`
    - Clicking a pill sets the query and triggers search
    - These render inside the same dropdown area

11. **Styling**: Use Tailwind classes. The search input should be prominent (large, rounded, with a search icon placeholder). Results dropdown has a white/light background with subtle shadow. Match the project's existing warm/serene aesthetic.

Do NOT fetch the index eagerly in useEffect on mount — only on first onFocus. Do NOT re-instantiate Fuse on every keystroke — store in useRef. Do NOT hardcode BASE_URL — always use the `baseUrl` prop.
  </action>
