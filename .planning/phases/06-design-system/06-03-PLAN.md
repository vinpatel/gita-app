3. Add `transition:animate="fade"` to the `<body>` tag to enable the default crossfade:
```html
<body class="min-h-screen flex flex-col bg-[--color-bg] text-[--color-text] transition-colors duration-300" transition:animate="fade">
```

**global.css changes — View Transition animation customization (per D-12):**

Append to global.css AFTER the prefers-reduced-motion block, BEFORE @layer utilities:

```css
/* === VIEW TRANSITION ANIMATIONS (per D-12, D-14) === */
/* Contemplative crossfade: 300ms, ease-out entrance, ease-in exit */
::view-transition-old(root) {
  animation: 300ms ease-in both fade-out;
}

::view-transition-new(root) {
  animation: 300ms ease-out both fade-in;
}

@keyframes fade-out {
  from { opacity: 1; }
  to { opacity: 0; }
}

/* fade-in already defined above — reused here */

/* Shared element morphing for chapter headers and verse refs */
/* Elements must add transition:name={`chapter-${n}`} in their template */

/* === HOVER/INTERACTIVE TRANSITIONS (per D-12) === */
@layer base {
  a, button, [role="button"] {
    transition: color 200ms ease-out, background-color 200ms ease-out, border-color 200ms ease-out, opacity 200ms ease-out;
  }
}
```

NOTE: The prefers-reduced-motion media query (already in global.css from Plan 01) automatically handles D-13 — it zeroes all animation and transition durations via the !important rule. No additional reduced-motion work needed for View Transitions because the browser's View Transitions API also respects prefers-reduced-motion natively.
